# Brain Schema - Page Format Standard

All pages in the brain follow this structure for consistency, searchability, and back-linking.

## Format

```markdown
# [Title]

[One-line executive summary]

## Current State

[What is true right now? Key facts, status, metrics.]

## Open Threads

- [Question 1]
- [Question 2]
- [Unresolved item]

---

## Timeline

**YYYY-MM-DD** | [Brief note or fact]. Source: [link or attribution].

**YYYY-MM-DD** | [Previous entry].
```

## Rules

1. **Title** — Clear, unique, searchable
2. **Executive summary** — One line; answerability check
3. **Current state** — The ground truth as of today
4. **Open threads** — Questions, decisions, unknowns
5. **Timeline** — Append-only facts with sources and dates
6. **Back-links** — Links to/from related pages (auto-generated by signal-detector)

## Naming Conventions

- **People:** `brain/people/FIRSTNAME-LASTNAME.md` (e.g., `john-doe.md`)
- **Companies:** `brain/companies/COMPANY-NAME.md` (e.g., `acme-corp.md`)
- **Deals:** `brain/deals/DEAL-NAME.md` (e.g., `series-a-funding.md`)
- **Meetings:** `brain/meetings/YYYY-MM-DD-TITLE.md` (e.g., `2026-04-18-product-sync.md`)
- **Concepts:** `brain/concepts/CONCEPT-NAME.md` (e.g., `vector-search.md`)
- **Ideas:** `brain/ideas/IDEA-NAME.md` (e.g., `chatbot-integration.md`)

All lowercase, dashes for spaces, no special characters.

## Consistency Matters

Search, enrichment, and dream cycles all depend on consistent formatting. Keep it simple, keep it standard.
